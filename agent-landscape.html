Interactive Coding Agent

Claude Code / Codex / Cursor

General-purpose coding agents you interact with in a terminal or IDE. They have built-in tools (file edit, bash, search) and a human-in-the-loop approving each step. The agent loop runs until the task is done or you intervene.

Human in loop: Always

Automated Coding Agent

Claude Code -p / GitHub Actions

Same general-purpose coding agents, but run headless via scripts or CI/CD. No human in the loop during execution. Triggered by events (PR opened, issue tagged). Still freeform — the agent decides what to do.

Workflow: Freeform
Human in loop: Before/after only
Determinism:
Customization:
Examples
claude -p "fix this" claude-code-action Codex --quiet Devin
General Assistant Platform

OpenClaw / Custom Assistants

Independent agent loop and tool system, not wrapping a coding agent. LLM-agnostic, connects to messaging platforms (Slack, WhatsApp, etc.). General-purpose but with its own architecture. Persistent memory across sessions.

Workflow: Freeform
Human in loop: Conversational
Determinism:
Customization:

Agent Framework / SDK

Claude Agent SDK / LangGraph / CrewAI

Libraries for building agents, not agents themselves. You define the tools, the loop structure, and the orchestration. The framework handles the LLM API calls and tool execution plumbing. You build whatever workflow you want on top.

Workflow: You define it
Human in loop: You decide

Customization:

Custom Domain-Specific System

AI-Native Hedge Fund (Your System)

Custom-built agent loop with domain-specific orchestration. Hard-coded phased pipeline (project → postmortem → refine → validate), policy-gated tool access, deterministic caching with cutoff dates, and self-improving skills tracked in SQLite. The workflow encodes financial domain logic — not a general agent, but a specific machine.

Which Approach Should You Use?

Are you doing a coding task (edit files, fix bugs, write tests)? Yes → Claude Code / Codex
↓ No
Do you want to automate coding tasks in CI/CD (PR review, test gen)? Yes → Claude Code -p / GitHub Actions
↓ No
Do you need a general assistant across messaging platforms? Yes → OpenClaw / similar
↓ No
Do you need a custom agent but want the LLM plumbing handled for you? Yes → Agent SDK / LangGraph
↓ No
Do you need a domain-specific system with deterministic workflows, custom tools, and self-improvement? Yes → Build your own (what you did)
